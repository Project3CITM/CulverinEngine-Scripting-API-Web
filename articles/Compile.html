Compile a CSharp Script

Download Library

Para empezar tenemos que instalarnos los prerequisitos para poder empezar, link: https://www.mono-project.com/docs/compiling-mono/windows/.

Si no te aclaras, tienes que descargarte la libreria y poner dentro del proyecto en una carpeta. Mira el gif:

Visual Studio Configuration

Ahora, tenemos que configurar nuestro proyecto para que podamos compilar un script desde el motor. Para ello vamos a segur una serie de pasos:

  • Seleccionamos el proyecto y abrimos la configuracion (Propiedades). Abrimos VC++ Direcories y alli tenemos que poner el path de la library de mono, utiliza el SolutionDir: $(SolutionDir)\Game\Mono\lib;$(LibraryPath). Con esto le decimos donde tendra la libreria.
  • Ahora tenemos que incluir en la ventana de C/C++ un directorio adicional: $(SolutionDir)\Game\Mono\include\mono-2.0

Una vez terminado lo anterior ya tenemos todo preparado para poder compilar los scripts.

Ahora vamonos al codigo y vamos a ver como podemos compilar un script. Para poder compilar un script utilizamos el comando "mcs" desde la consola "System". Puedes seguir el tutorial de mono que mas o menos lo entenderas, o puedes ver en la fotografia como lo hemos hecho nosotros. Es importante enterner esto, lo que hace el compilar un script es ver si hay algun error y tambien te generara una dll del script, si es asi te devolvera un 1 o si no tiene ningun error un 0.

Primero en la funcion le pasamos 3 argumentos:

  • const char* file: es el path del script .cs donde lo tengamos guardado.
  • std::string& libraryScript: esta string nos devolvera el path a library donde se importara el script una vez compilado. Más adelante explicare para que necesitamos esto.
  • const char* uid: es para poder recompilar los scripts, nosotros guardamos los scripts con su UID y si un script se ha editado queremos que se reimporte con el mismo uid por eso lo pasamos.

Dentro de la funcion lo que hacemos es coger correctamente los path para luego ponerlos en la funcion system(), lo que hacemos es primer seteamos la variable command a que utilice el programa mcs.exe que esta en las carpetas que antes hemos puesto en el motor. Le decimos que el target es library (dll) y el path de salida (out). Lo siguiente es por si utilizamos una Assembly Reference que ya llegaremos a eso más adelante.
