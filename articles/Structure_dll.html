Structure

Para la estructura empezamos haciendo los scripts sin que heredarán de nadie, entonces para acceder al GameObject del script al cual está asignado teníamos que cogerlo con "GameObject.gameObject". Luego vimos que este método es muy poco práctico y lo cambiamos:

New Structure: "ScriptName"::CulverinBehaviour::Component::Object.

CulverinBehaviour

Cogimos como referencia la estructura de scripting que utiliza Unity. Como padre de todos los script de nuestro gameplay "CulverinBehaviour" que tiene las funciones básicas (set enabled GetEnables, etc...). Es importante utilizar el namespace!

Este hereda de Component, que tiene un gameObject que es el padre del componente que estas utilizando (el propio script lo tratamos como un componente). A parte del gameObject, tiene más funciones como GetComponent() o transform. Todos los componentes que creemos heredarán de CulverinBehaviour ya que este hereda de Component.

Por último, Componente hereda de Object que es el que define todos los diferentes objetos que tendremos en el Engine (Components, GameObjects, etc...).
